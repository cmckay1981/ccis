# CCIS CRM Courses - Complete Summary

## Overview
Five Crew Resource Management (CRM) courses designed for high-reliability operations. Each course follows a modular structure with short, operational lessons delivered step-by-step through the CCIS training engine.

---

## Course 1: Decision Making (DM)

### Course Summary
Focuses on operational decision-making under pressure, uncertainty, and time constraints. Covers decision models, cognitive biases, and team decision processes.

### Module Structure
1. **Decision Fundamentals** (3 lessons)
   - Types of decisions: routine, tactical, critical
   - Speed vs quality trade-offs
   - Decision authority clarity

2. **Decision Models** (4 lessons)
   - OODA Loop (Observe, Orient, Decide, Act)
   - Recognition-Primed Decision Making
   - Decision trees for procedures
   - Naturalistic Decision Making in field conditions

3. **Cognitive Biases** (4 lessons)
   - Confirmation bias
   - Anchoring effect
   - Normalcy bias
   - Outcome bias

4. **Deciding Under Uncertainty** (4 lessons)
   - Probabilistic thinking
   - Worst-case analysis
   - Reversible vs irreversible decisions
   - Decision triggers

5. **Team Decision Making** (4 lessons)
   - Command vs consensus
   - Groupthink and dissent
   - Decision ownership
   - Pre-brief protocols

6. **Decisions Under Stress** (4 lessons)
   - Stress degradation effects
   - Fatigue impact on judgment
   - Tactical breathing techniques
   - Decision fatigue recognition

**Total Lessons:** 23
**Difficulty Progression:** Foundational → Models → Cognitive Challenges → Practical Application
**Recommended Duration:** 3-4 weeks

## Instructional Design Features

### Lesson Structure
- **Short Content:** 2-4 sentences per lesson
- **Questions:** 1-2 open-ended reflection questions
- **Optional MCQ:** Multiple choice with explanation
- **Optional Scenario:** Situational application

### Difficulty Progression
Each course follows:
1. **Introduce:** Basic concepts and definitions
2. **Apply:** Practical techniques and tools
3. **Challenge:** Complex scenarios and edge cases
4. **Integrate:** System-level and organizational perspectives

### Instructor Behavior (CCIS Engine)
- Deliver ONE lesson at a time
- Wait for user response before proceeding
- Accept only A/B/C/D for MCQs
- Respond "Confirmed" or "Incorrect. Recalibrate..." for MCQ answers
- Track progress in user-progress collection
- Resume capability via user profile memory
